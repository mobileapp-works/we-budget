# 要件定義書

## 1. アプリ概要
- アプリ名: WeBudget（ストア表示名。Bundle ID: com.mobileappworks.webudget）
- ジャンル: ライフスタイル / ファイナンス
- コンセプト: カップル（2人）の支出を記録し、立替状況と精算額を一目で把握できる家計簿アプリ。レシート撮影で簡単に記録、固定費は自動計算、変動固定費は入力リマインドで漏れを防ぐ。
- 参考アプリ: OsidOri（カップル家計簿）、B/43（ペア口座）、Zaim（レシート撮影）、Splitwise（割り勘・立替精算）

## 2. ターゲットユーザー
- ペルソナ: 同棲・非同棲・夫婦を問わず、2人でお金の管理をしたいカップル全般
- 利用シーン: 買い物後のレシート撮影、月末の精算確認、家計の振り返り
- 技術リテラシー: 一般的なスマホユーザー（LINE・Instagramを普通に使えるレベル）

## 3. 解決する課題
- ユーザーの課題:
  1. 誰がいくら立て替えたかが曖昧になり、精算時に揉める
  2. 結局いくら精算すればいいのかすぐにわからない
  3. 今月いくら使ったか・何に使ったかが見えない
  4. 家賃など毎月同じ金額の支出を毎回入力するのが面倒
  5. 光熱費など毎月ある支出の入力を忘れる
  6. 予算オーバーに気づかず使いすぎてしまう
- 既存の解決策: Excel / メモアプリ / 個人向け家計簿アプリ / LINEで報告し合う
- 本アプリの優位性:
  - レシート撮影（OCR）で入力の手間を最小化
  - 支払い者（自分 / パートナー / 共同口座）を記録し、立替残高を自動計算
  - 固定費の自動計上＋変動固定費のリマインドで入力漏れを防止
  - カテゴリ別のグラフで支出を見える化
  - 予算設定＋超過アラートで使いすぎを防止

## 4. 技術構成
- 対応OS: iOS（将来Android対応）
- フレームワーク: React Native（Expo / Expo Router）
- 言語: TypeScript（strict モード）
- バックエンド: Supabase（Auth / Database / Storage）
- ストレージ: Supabase Storage（レシート画像保存）
- 状態管理: useState / Zustand / TanStack Query
- 広告SDK: Google AdMob（バナー広告）
- 監視: Sentry
- 多言語: i18next（日本語・英語）
- ビルド・配信: EAS Build / EAS Submit
- 認証: あり（Supabase Auth）→ アカウント削除も実装必須
  - ログイン方法: メールアドレス + Apple Sign-In + Google Sign-In
- 外部AI利用: あり → 提供元名 + 同意画面が必須
  - OCR: Google Cloud Vision API（レシート読み取り）
  - 将来: 支出分析・カテゴリ自動分類などのAI活用を検討
- プッシュ通知: Expo Notifications
- オフライン対応: なし（オンライン接続必須）
- その他SDK: なし

## 5. 収益化方針
- 収益モデル: フリーミアム（無料 + 広告 / 課金で広告非表示 + AI機能解放）
- 広告種別: バナー広告（画面下部に常時表示）
- 広告非表示オプション: 課金（サブスクリプションまたは買い切り、詳細は07_monetizationで決定）
- 課金特典:
  - 広告非表示
  - AI機能の利用（支出分析・アドバイス等）
- 価格帯: 07_monetizationフェーズで決定

## 7. ビジネスルール

### 7-1. 立替精算ルール
- 負担割合はカップル単位で設定（デフォルト 50:50、例: 60:40 や収入比も可）
- 「自分が払った」「パートナーが払った」「共同口座から払った」の3種類を記録
- 立替残高 = 各自が立て替えた合計 − 負担割合で算出した本来の負担額
- 精算は好きなタイミングで実行可能（精算ボタン → 残高リセット）
- 月末に未精算がある場合はリマインド通知
- 精算履歴は記録として残す

### 7-2. 固定費ルール
- **固定費（金額固定）**: 毎月同じ日に同じ金額を自動で支出に計上（例: 家賃）
- **変動固定費（金額変動）**: 毎月の入力期限を設定し、未入力ならリマインド（例: 光熱費、水道代）
- 支払い者（自分/パートナー/共同口座）も固定費ごとに設定

### 7-3. 予算ルール
- カテゴリ別または全体で月間予算を設定
- 予算の80%到達時に警告アラート、100%超過時に超過アラート
- アラートはプッシュ通知＋アプリ内通知の両方

### 7-4. 共同口座ルール
- 共同口座への入金（各自がいくら入れたか）を記録
- 共同口座からの支出は立替精算の対象外（既に共同資金から支払済みのため）
- 共同口座の残高を表示

### 7-5. サブスク課金時ヒアリング（将来機能）
- サブスク加入直後にオンボーディングとして以下をヒアリング:
  - 各自の年収（税込）/ 手取り月収
  - 貯蓄目標額（月額 or 年額）
  - ライフステージ（同棲 / 婚約 / 新婚 / 子育て中 等）
  - 負担割合の希望（折半 / 収入比 / カスタム）
- ヒアリング結果はAI分析の入力データとして利用
  - 年収比での適正負担割合の提案
  - 支出が手取りの何%かの可視化
  - 貯蓄目標に対する達成度シミュレーション
- 入力は任意（スキップ可能）、後から設定画面で変更可能
- 収入情報はパートナーには非公開（AI分析の結果のみ共有する設計も検討）

### 7-6. 通知ルール
- **支出記録通知**: パートナーが支出を記録したらプッシュ通知（「〇〇さんが食費 ¥2,000を記録しました」）
- **精算通知**: 精算が実行されたら両者にプッシュ通知
- **変動固定費リマインド**: 入力期限を過ぎたらプッシュ通知＋アプリ内通知
- **予算アラート**: 80%到達・100%超過時にプッシュ通知＋アプリ内通知
- **月末精算リマインド**: 未精算の立替があれば月末にプッシュ通知
- 通知の種類ごとにON/OFFを設定画面で切り替え可能

### 7-7. 支出の編集・削除ルール
- 支出の編集はどちらのユーザーでも可能（自分・パートナーどちらが入力したものでも）
- 支出の削除もどちらのユーザーでも可能
- 編集・削除したら相手にプッシュ通知で知らせる

### 7-8. OCRルール
- レシート撮影後、読み取り結果（金額・店名・日付）をプレビュー表示
- ユーザーが確認・修正してから保存（自動保存しない）
- 読み取り精度が低い項目は未入力状態で表示し、手動入力を促す
- レシート画像は支出レコードに紐づけてSupabase Storageに保存（後から閲覧可能）

### 7-9. 通貨ルール
- 支出ごとに通貨を選択可能（デフォルトはJPY）
- レポート・精算時の表示通貨はJPY（為替レートは手動入力 or 自動取得を検討）

### 7-10. ペア解除時のデータ
- ペアを解除しても過去の支出データは各自のアカウントに残る（自分が記録した分は閲覧可能）
- 未精算の立替があればペア解除前に精算を促す
