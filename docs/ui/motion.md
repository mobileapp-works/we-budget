# モーション定義（WE BUDGET）

`_defaults` を踏襲。過度な動きを避け、自然で素早い体感にする。ポップさは「ほんの少しの弾み」で表現し、やりすぎない。

## デュレーション
| トークン | 値 | 用途 |
|---------|-----|------|
| `instant` | 100ms | 押下フィードバック |
| `fast` | 200ms | トグル・チップ選択・小さな状態変化 |
| `normal` | 300ms | 画面遷移・モーダル表示 |
| `slow` | 450ms | 大きなレイアウト変化 |

## イージング
- 標準: ease-in-out（自然な加減速）。出現: ease-out / 退場: ease-in。
- **ブランドの遊び**: 支出保存成功・精算完了など達成感のある瞬間のみ、軽いスケール/バウンス（控えめ）を許可。多用しない。

## 具体例
- FAB押下 → 支出入力モーダルは下からスライドイン（`normal`）。
- 保存成功時のチェックマーク: `fast` でフェード＋わずかにスケール。
- 予算バーの進捗: 値変化時に `normal` でアニメート（0→現在値）。

## 実装方針
- `react-native-reanimated` を基本に、ネイティブスレッドで動かす（カクつき防止）。
- 画面遷移は Expo Router / React Navigation の標準トランジションを尊重する。
- **「視差効果を減らす」「アニメーションを減らす」設定に追従**（`AccessibilityInfo.isReduceMotionEnabled()` がオンならフェードのみ/動き最小化）。
- 60fpsを維持できない重いアニメーションは入れない。
