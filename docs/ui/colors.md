# カラー定義（WE BUDGET）

`_defaults` のパレットをベースに、**ブランドカラーをコーラル（メイン）＋ティール（アクセント）**へ差し替えた確定ドラフト。
すべての文字色・背景色の組み合わせは **WCAG AA（4.5:1）以上**を満たす。Claude Design で微調整した場合は再確認すること。

## ブランドカラー
| トークン | ライト | ダーク | 用途 | コントラスト確認 |
|---------|--------|--------|------|----------------|
| `primary` | `#C8472F` | `#C8472F` | メインアクション、選択状態（深めのコーラル） | white文字 4.8:1 ✅ |
| `primaryText` | `#FFFFFF` | `#FFFFFF` | primary上の文字 | AA合格 |
| `accent` | `#0F766E` | `#2DD4BF` | 強調・バッジ・リンク（ティール） | 下記参照 |
| `accentText` | `#FFFFFF` | `#0F2A27` | accent上の文字 | AA合格 |
| `coralTint` | `#FF6F5C` | `#FF7A66` | 装飾・イラスト・選択ハイライト（白文字バッジには使わない） | 装飾用 |
| `coralSoft` | `#FFE7E0` | `#3A211B` | コーラルの淡い面（チップ背景・進捗トラック） | 上に textPrimary |

> **コーラル運用ルール**: 白文字を載せるボタン等は必ず深いコーラル `primary (#C8472F)` を使う。鮮やかな `coralTint (#FF6F5C)` はイラスト・アイコン・選択ハイライト用（白文字背景には使わない＝コントラスト不足のため）。
> **ティール運用ルール**: ライトでテキスト/バッジに使う場合は `accent (#0F766E, teal-700)` を白文字で（AA合格）。ダークは `#2DD4BF` を暗い文字で。

## お金の表現（家計簿固有）
| 状況 | 色 | 補助 |
|------|-----|------|
| 自分が受け取る / プラス残高 | `success` | 「＋」記号＋アイコン |
| 自分が支払う / マイナス残高 | `textPrimary`（中立）／強調時 `primary` | 「−」記号＋アイコン |
| 予算 〜79% | `success` | 進捗バー |
| 予算 80〜99% | `warning` | 進捗バー＋警告アイコン |
| 予算 100%超 | `error` | 進捗バー＋超過アイコン |

> **色だけで損益を伝えない**: プラス/マイナスは必ず記号（＋/−）やアイコンを併用する。

## 実装方針
- ライト/ダークの両定義を必ず持ち、`useColorScheme()` で切り替える。
- 色は直値で書かず、必ずトークン経由（`constants/colors.ts`）で参照する。
- 差し替え・微調整時は **対 white / 対 textPrimary のコントラストを再確認**（4.5:1以上）。
