# スペーシング定義（WE BUDGET）

`_defaults` の **8ptグリッド**を踏襲。ポップ＆あたたかい世界観に合わせ、**角丸をやや大きめ**にして柔らかい印象にする（4pxは微調整のみ）。

## スペーシングスケール
| トークン | 値 | 用途 |
|---------|-----|------|
| `xxs` | 4px | アイコンとラベルの極小間隔 |
| `xs` | 8px | 関連要素間 |
| `sm` | 12px | リスト項目内の間隔 |
| `md` | 16px | 標準の要素間・画面左右マージン |
| `lg` | 24px | セクション間 |
| `xl` | 32px | 大きな区切り |
| `xxl` | 48px | 画面上下の大余白 |

## 角丸（radius）※デフォルトより気持ち大きめ（柔らかさ）
| トークン | 値 | 用途 |
|---------|-----|------|
| `radiusSm` | 10px | ボタン・入力欄 |
| `radiusMd` | 16px | カード |
| `radiusLg` | 20px | モーダル・シート |
| `radiusFull` | 9999px | カプセル型・チップ・アバター |

## レイアウト基準
- 画面左右マージン: `md`（16px）。
- タップ可能要素の最小サイズ: **44×44pt**（余白で稼いでよい）。
- カード間の縦間隔: `sm`〜`md`。セクション間は `lg`。
- iPad ではコンテンツ最大幅 700px・中央寄せ。リストは2カラムグリッド。
- バナー広告（高さ50-60px）分を画面下部に確保し、最下部のコンテンツが隠れないようにする。

## 実装方針
- 直値の余白指定を禁止。`constants/spacing.ts` のトークンを参照する。
- マージンは「外側の余白は親が決める」原則（子コンポーネントに外側marginを持たせない）。
