# コンポーネント定義（WE BUDGET）

`_defaults` をベースに、コーラル/ティールのブランドと家計簿固有のコンポーネントを追加。
`components/` に共通実装し全画面で使い回す。すべての操作可能要素に `accessibilityRole` / `accessibilityLabel` を付ける（→ `accessibility.md`）。

## ボタン
| 種別 | 背景 | 文字 | 高さ | 角丸 |
|------|------|------|------|------|
| Primary | `primary` | `primaryText` | 48px | `radiusSm` |
| Secondary | `surface` + `border` | `textPrimary` | 48px | `radiusSm` |
| Accent | `accent` | `accentText` | 48px | `radiusSm` |
| Destructive | `error` | `#FFFFFF` | 48px | `radiusSm` |
| Text | 透明 | `primary` | 44px | - |

- 無効状態: 透明度 0.4 + 操作不可。押下中はわずかに暗く（opacity 0.7）。
- 非同期処理中はスピナー表示 + 二度押し防止。
- `accessibilityRole="button"`、`accessibilityState={{ disabled, busy }}` を付与。
- FAB（支出入力の「＋」）: `primary` 背景・円形（`radiusFull`）・サイズ56px・影付き。タブ中央配置。

## カード
- 背景 `surface`、角丸 `radiusMd`、内側パディング `md`。
- 影は控えめ（iOS: 薄いshadow / Android: elevation 2）。境界は `border` でも可。
- **サマリーカード**（ホーム上部）: 金額を `display`/`title1` Bold。背景に `coralSoft` を敷いてブランド感を出してよい。

## 立替残高カード（家計簿固有）
- 「誰が誰にいくら」を1行で明示（例: 「あなた → ◯◯さん ¥3,200」）。
- 受け取り側はプラス＝`success`、支払い側はマイナス＝中立色 + 「−」記号。色だけに頼らず記号併用。
- カードタップで精算画面へ遷移（`accessibilityHint`「精算画面を開く」）。
- ソロモードでは非表示（代わりに「パートナーを招待」CTA）。

## 予算進捗バー（家計簿固有）
- トラックは `coralSoft`、塗りは使用率で色分け: 〜79% `success` / 80〜99% `warning` / 100%超 `error`。
- バー横に「使用額 / 予算額」とパーセントをテキスト表示（色だけに頼らない）。
- バータップで予算設定画面へ。

## チップ / バッジ
- カテゴリチップ: `radiusFull`、カテゴリカラーの淡色背景 + 濃色文字（AA確認）。アイコン併記。
- カテゴリフィルタの選択中: `primary` 背景 + `primaryText`。
- 未読バッジ（通知ベル）: `error` の小ドット + 件数。

## 入力フィールド
- 背景 `surfaceElevated`、ボーダー `border`、角丸 `radiusSm`、高さ48px。
- フォーカス時はボーダーを `primary` に。エラー時は `error` + エラーメッセージを下に表示。
- ラベルは常時表示（プレースホルダーをラベル代わりにしない）。
- 金額入力: 数値キーボード、通貨記号を前置、桁区切り自動。
- 支払い者セレクタ: 「自分 / 相手 / 共同口座」のセグメント（ソロモードでは「自分」固定で非表示）。

## リストアイテム
- 最小高さ44px、左右パディング `md`、区切り線 `border`。
- 支出リスト行: 左にカテゴリアイコン（丸背景）、中央に店名/メモ、右に金額＋支払い者の小ラベル。
- タップで支出詳細へ（`accessibilityRole="button"` + 押下フィードバック）。
- 大量データは FlatList（または FlashList）を使う。

## アイコン
- サイズ: 小16 / 中24 / 大32px。**塗り（filled）系で統一**してポップな印象に。
- 単体アイコンボタンには必ず `accessibilityLabel`。
- カテゴリアイコンは丸背景 + カテゴリカラーで視認性を確保。

## フィードバック要素（全アプリ用意 + 本アプリの文言）
- **Toast**: 「支出を保存しました」「精算が完了しました」等。成功=`success`/エラー=`error`、アイコン併用。
- **確認ダイアログ**: 削除・精算・ペア解除・アカウント削除の前に必ず表示。精算は金額を明記。
- **空状態（Empty）**: やわらかいイラスト/アイコン + 説明 + CTA。例: ホーム「最初の支出を記録しよう」。
- **ローディング**: スケルトン優先（カード/リスト形状）。難しければスピナー。
- **エラー状態**: 原因 + 「再試行」ボタン。無言で失敗させない。

## 4状態の対応（全画面共通）
| 状態 | 見せ方 |
|------|--------|
| Loading | スケルトン（カード/リストの形） |
| Empty | イラスト + 説明 + CTAボタン |
| Error | 原因テキスト + 「再試行」 |
| Success | 通常表示。操作後はToastでフィードバック |
