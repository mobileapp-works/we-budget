1. アプリ概要

アプリ名
WeBudget（Bundle ID: com.mobileappworks.webudget）
ジャンル
ライフスタイル / ファイナンス
コンセプト
カップル（2人）の支出を記録し、立替状況と精算額を一目で把握できる家計簿アプリ。レシート撮影で簡単に記録、固定費は自動計算、変動固定費は入力リマインドで漏れを防ぐ。
参考アプリ
OsidOri（カップル家計簿）、B/43（ペア口座）、Zaim（レシート撮影）、Splitwise（割り勘・立替精算）

2. ターゲットユーザー

ペルソナ
同棲・非同棲・夫婦を問わず、2人でお金の管理をしたいカップル全般
利用シーン
買い物後のレシート撮影、月末の精算確認、家計の振り返り
技術リテラシー
一般的なスマホユーザー（LINE・Instagramを普通に使えるレベル）

3. 解決する課題

ユーザーの課題
  1. 誰がいくら立て替えたかが曖昧になり、精算時に揉める
  2. 結局いくら精算すればいいのかすぐにわからない
  3. 今月いくら使ったか・何に使ったかが見えない
  4. 家賃など毎月同じ金額の支出を毎回入力するのが面倒
  5. 光熱費など毎月ある支出の入力を忘れる
  6. 予算オーバーに気づかず使いすぎてしまう
既存の解決策
Excel / メモアプリ / 個人向け家計簿アプリ / LINEで報告し合う
本アプリの優位性
  • レシート撮影（OCR）で入力の手間を最小化
  • 支払い者（自分 / パートナー / 共同口座）を記録し、立替残高を自動計算
  • 固定費の自動計上＋変動固定費のリマインドで入力漏れを防止
  • カテゴリ別のグラフで支出を見える化
  • 予算設定＋超過アラートで使いすぎを防止

7. ビジネスルール

7-1. 立替精算ルール

負担割合
カップル単位で設定（デフォルト 50:50、例: 60:40 や収入比も可）
支払い種別
「自分が払った」「パートナーが払った」「共同口座から払った」の3種類
計算方法
立替残高 = 各自が立て替えた合計 − 負担割合で算出した本来の負担額
精算タイミング
好きなタイミングで実行可能（精算ボタン → 残高リセット）。月末に未精算がある場合はリマインド通知
精算方法
アプリは精算額を表示するのみ。実際の支払いはアプリ外（現金・PayPay等）で行う
精算履歴
記録として残す

7-2. 固定費ルール

固定費（金額固定）
毎月同じ日に同じ金額を自動で支出に計上（例: 家賃）
変動固定費（金額変動）
毎月の入力期限を設定し、未入力ならリマインド（例: 光熱費、水道代）
支払い者設定
固定費ごとに支払い者（自分/パートナー/共同口座）を設定

7-3. 予算ルール

設定単位
カテゴリ別または全体で月間予算を設定
アラート
80%到達時に警告アラート、100%超過時に超過アラート
通知方法
プッシュ通知＋アプリ内通知の両方

7-4. 共同口座ルール

入金記録
各自がいくら入れたかを記録
精算対象
共同口座からの支出は立替精算の対象外（既に共同資金から支払済み）
残高表示
共同口座の残高を表示

7-5. 通知ルール

支出記録通知
パートナーが支出を記録したらプッシュ通知
精算通知
精算が実行されたら両者にプッシュ通知
変動固定費リマインド
入力期限を過ぎたらプッシュ通知＋アプリ内通知
予算アラート
80%到達・100%超過時にプッシュ通知＋アプリ内通知
月末精算リマインド
未精算の立替があれば月末にプッシュ通知
通知設定
種類ごとにON/OFFを設定画面で切り替え可能

7-6. 編集・削除ルール

編集権限
どちらのユーザーでも編集・削除可能（自分・パートナーどちらが入力したものでも）
変更通知
編集・削除したら相手にプッシュ通知で知らせる

7-7. OCRルール

読み取りフロー
レシート撮影 → 読み取り結果（金額・店名・日付）をプレビュー表示 → ユーザーが確認・修正 → 保存
読み取り失敗時
精度が低い項目は未入力状態で表示し、手動修正を促す
画像保存
レシート画像は支出レコードに紐づけてSupabase Storageに保存（後から閲覧可能）

7-8. 通貨ルール

通貨選択
支出ごとに通貨を選択可能（デフォルトはJPY）
表示通貨
レポート・精算時の表示通貨はJPY（為替レートは手動入力 or 自動取得を検討）

7-9. サブスク課金時ヒアリング（将来機能）

ヒアリング項目
各自の年収（税込）/ 手取り月収 / 貯蓄目標額 / ライフステージ / 負担割合の希望
AI分析への活用
年収比での適正負担割合の提案 / 支出が手取りの何%かの可視化 / 貯蓄目標に対する達成度シミュレーション
入力ルール
任意（スキップ可能）、後から設定画面で変更可能
プライバシー
収入情報はパートナーには非公開（AI分析の結果のみ共有する設計も検討）

7-10. ペア解除時のデータ

データ保持
ペアを解除しても過去の支出データは各自のアカウントに残る（自分が記録した分は閲覧可能）
未精算の扱い
未精算の立替があればペア解除前に精算を促す

1. App Overview

App Name
WeBudget (Bundle ID: com.mobileappworks.webudget)
Genre
Lifestyle / Finance
Concept
A shared household budget app for couples. Track expenses, see who paid what, auto-calculate reimbursements, and visualize spending by category. Snap receipts for easy logging, auto-record fixed costs, and get reminders for recurring variable bills.
Reference Apps
OsidOri (couple budgeting), B/43 (shared account), Zaim (receipt scanning), Splitwise (expense splitting)

2. Target Users

Persona
All couples wanting to manage money together — cohabiting, dating, or married
Use Cases
Scanning receipts after shopping, checking settlement at month-end, reviewing household spending
Technical Literacy
Average smartphone user (comfortable with LINE, Instagram)

3. Problem to Solve

User's Problem
  1. Who paid how much becomes unclear, causing disputes at settlement time
  2. Hard to know how much to settle at a glance
  3. Can't see how much was spent this month or on what
  4. Tedious to manually enter fixed costs (rent, etc.) every month
  5. Forget to log recurring variable bills (utilities, etc.)
  6. Overspend without realizing the budget is exceeded
Existing Solutions
Excel / note apps / personal budgeting apps / reporting via LINE
Our Advantage
  • Receipt OCR minimizes manual input
  • Track payer (self / partner / shared account) and auto-calculate reimbursement
  • Auto-record fixed costs + reminders for variable recurring bills
  • Visualize spending by category with charts
  • Budget alerts to prevent overspending
